# OptimaLearn — AI Study Planner Redesign


## Overview

This project investigates why CAT aspirants drop off from OptimaLearn's AI Study Planner before completing onboarding — and proposes a redesigned onboarding flow and UI to fix it.

The core finding: users arrive overwhelmed and looking for clarity. The current planner recreates that overwhelm instead of resolving it.

---

## Problem Statement

> *"Users abandon the AI study planner not because they don't want a plan — but because onboarding asks questions they can't answer, produces a plan they don't understand, and presents a UI that feels cluttered rather than actionable."*

**Two distinct failure layers were identified:**

| Layer | Failure Point | Fix Direction |
|-------|--------------|---------------|
| 1 | Onboarding questionnaire — abstract, confusing questions | Reframe in user language + add progress indicators |
| 2 | Planner output — dense, no clear entry point | Surface a single "your next task" card prominently |

---

## Research Evidence

| Method | Sample | Key Signal |
|--------|--------|------------|
| User Research Interviews | 17 users | Planning overhead cited as a top pain point across all prep stages |
| Usability Test Sessions | 6 users | Study planner was the **most-mentioned pain point** across all sessions |

**Severity:** High — affects the core value proposition, causes first-session dropout, and is fixable without backend changes.

---

## Key Insight

Users came to OptimaLearn *because* they were overwhelmed preparing for CAT. The onboarding should have been their first moment of relief — instead, it recreated exactly the feeling they were trying to escape.

The fix is not fewer features. It's a simpler entry point: one clear next action, questions in plain language, and a plan that feels like a guide — not a system to navigate.
